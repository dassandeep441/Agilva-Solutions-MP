QR Hub for Jira

Trust
Centre

Security · Privacy · Compliance · Last updated: June 2026

🏗

Runs on Atlassian

QR Hub for Jira runs entirely on Atlassian Forge infrastructure. There are no external servers, no external APIs, and no egress of any kind. All computation happens inside Atlassian’s platform.

🔒

No Data Leaves Atlassian

The only data persisted by QR Hub is the dashboard gadget URL, stored via storage:app within Atlassian Forge. No user data, issue content, or URLs are transmitted to any Agilva-controlled server.

🌎

Data Residency

QR Hub inherits Atlassian’s full data residency guarantees. Data is stored in the region selected for your Atlassian Cloud instance. Agilva has no control over or access to this data.

📋

GDPR & SOC 2

QR Hub does not independently hold GDPR or SOC 2 certifications. Compliance is inherited from the Atlassian Forge platform, which operates under Atlassian’s own certifications and DPA.

Scopes requested

QR Hub for Jira requests only the minimum Atlassian scopes required to operate each feature. No scope is requested beyond what is functionally necessary.

Scope Reason
read:jira-work Read issue details, board and project information to display alongside QR codes in the Issue QR Panel.
read:board-scope:jira-software Read board name and ID for board QR generation from the board & backlog menus.
read:project:jira Read project details for project context in the QR panel and print layouts.
read:issue-details:jira Read extended issue fields (priority, reporter, description, dates) for the Full Issue Details and Jira Tag PDF print layouts.
storage:app Persist the dashboard gadget URL across page refreshes using Forge Storage inside Atlassian infrastructure.

Certifications & badges

  • Runs on Atlassian — Qualifies for the “Runs on Atlassian” badge. No external hosting, no external APIs, no data egress.
  • Cloud Fortified — Not yet applied for. Agilva will apply for Cloud Fortified designation after the Marketplace listing is live.
  • GDPR / SOC 2 — Inherited from Atlassian Forge. No independent certification held by Agilva at this time.

Jira permission model

QR Hub does not modify, bypass, or extend Jira’s permission system in any way. Generated QR codes contain only public Jira URLs. When a QR code is scanned, Jira’s standard access controls apply — users who do not have access to an issue or board will be handled by Jira as normal.

Vulnerability disclosure

If you discover a security issue in QR Hub for Jira or any Agilva product, please report it responsibly by emailing security@agilva.com. We acknowledge all reports within 48 hours and aim to resolve critical issues within 14 days.

Questions?

For any security or compliance questions, contact us directly. We respond within one business day.
